# Class-to-Lab Notes - Lab 02

## What was studied

This class covered RACF security event logging and auditing concepts.

The practical command used was:

```text
SETROPTS LIST
```

## Why this matters

`SETROPTS LIST` is useful because it shows RACF global security options. These options influence how RACF behaves across the system.

For audit and security review work, this helps identify:

- Active RACF classes.
- Generic profile support.
- Global checking classes.
- Dataset protection indicators.
- Password policy indicators.
- User revocation settings.
- Security retention settings.

## How the screenshots were used

The screenshots collected during the class were converted into GitHub evidence.

Each screenshot is paired with:

1. The command executed.
2. The relevant RACF output.
3. A security interpretation.
4. A production-oriented observation.

## What this lab proves to recruiters

This lab proves that the candidate can work with real 3270/RACF screens, execute a native RACF inquiry command, and translate raw mainframe security output into audit-style findings.
