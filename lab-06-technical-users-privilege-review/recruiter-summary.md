# Recruiter Summary

This lab demonstrates hands-on RACF security review skills on z/OS:

- audited technical user IDs used by started tasks
- identified privileged attributes such as `SPECIAL`, `OPERATIONS`, and `PROTECTED`
- reviewed the central `SYS1` RACF group
- distinguished between lab defaults and production hardening expectations
- documented evidence in a structured GitHub-ready lab

The strongest portfolio value is the ability to connect runtime started-task ownership from SDSF with RACF user and group definitions.
